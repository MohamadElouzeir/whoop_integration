Integration Depth

Exactly how WHOOP data
drives Taqa

Every piece of WHOOP data we consume is surfaced directly to the user and their coach — no black boxes.

📊

TAQA Fitness Score

Our composite daily score (0–100) draws from whichever wearable the user has connected. When WHOOP is connected, recovery and strain are used as inputs. The app labels the data source so users always know where each score comes from.

recovery_score strain hrv_rmssd total_sleep_minutes
💤

Sleep Intelligence

Users track total sleep, time in bed, efficiency, sleep stages (Light / Deep / REM), disturbances, and sleep cycles — all sourced from WHOOP — with weekly, monthly, and yearly trend charts.

total_sleep_min sleep_stages efficiency disturbances
❤️

Recovery Details

A dedicated Recovery screen shows HRV (RMSSD), resting heart rate, SpO₂, skin temperature, and a 7-day recovery trend line — all rendered from WHOOP's daily recovery payload.

hrv_rmssd resting_hr spo2 skin_temp_c
⚡

Daily Strain & Cycle

Strain (0–21 scale), average and max heart rate, and total energy expenditure are displayed per day with a 7-day HR trend. Strain feeds our ACWR (acute:chronic workload ratio) engine.

strain avg_hr max_hr energy_kj
🏋️

Coach Analytics View

Coaches see their clients' 7-day WHOOP recovery and strain charts side by side, with connection status and the ability to navigate through historical weeks — enabling data-driven coaching decisions.

7-day recovery 7-day strain connection status
🔗

User-Initiated OAuth

Users connect WHOOP from the app's Settings screen via a single-tap "Connect Whoop" flow. Only users who explicitly authorize are synced. We never access WHOOP data without consent.

OAuth 2.0 user consent revocable

Technical Architecture

How WHOOP data flows
through Taqa

WHOOP data is fetched daily per user and normalized into our unified scoring engine alongside Fitbit and Apple HealthKit.

⌚

WHOOP Device

User wears WHOOP. Data syncs to WHOOP cloud.

→
🔐

WHOOP OAuth

User authorizes Taqa. Token stored securely per user.

→
⚙️

Taqa Backend

Daily sync job fetches & normalizes WHOOP payload per user.

→
🧮

Scoring Engine

Computes Sleep, Recovery, Stress & Training Load scores.

→
📱

User & Coach

Scores, trends, and raw metrics surfaced in app in real time.

WHOOP API Endpoints We Use
GET
/v1/cycle
Daily strain, avg/max HR, energy expenditure
GET
/v1/recovery
Recovery score, HRV RMSSD, resting HR, SpO₂, skin temp
GET
/v1/sleep
Total sleep, time in bed, stages (light/deep/REM), efficiency, disturbances
GET
/v1/user/measurement/body
Weight used to supplement user profile if not entered manually
POST
/oauth2/token
Token refresh — runs automatically before each daily sync
GET
/v1/user/profile/basic
Verify token validity and fetch linked user profile on connect
